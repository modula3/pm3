Dbc.count

 

import com.sleepycat.db.*;


public int count(int flags)
    throws DbException;

Description

The Dbc.count method returns a count of the number of duplicate data items for the key to which the cursor refers. If the underlying database does not support duplicate data items, the call will still succeed and a count of 1 will be returned.

The flags parameter is currently unused, and must be set to 0.

If the cursor argument is not yet initialized, the Dbc.count method throws an exception that encapsulates Db.EINVAL. Otherwise, the Dbc.count method throws an exception that encapsulates a non-zero error value on failure.

Errors

The Dbc.count method may fail and throw an exception for errors specified for other Berkeley DB and C library or system methods. If a catastrophic error has occurred, the Dbc.count method may fail and throw a DbRunRecoveryException, in which case all subsequent Berkeley DB calls will fail in the same way.
